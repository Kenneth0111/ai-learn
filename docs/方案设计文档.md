# 方案设计文档 — AI学习闯关小程序

## 一、总览

采用 **Taro 4 (React/TypeScript)** + **FastAPI (Python)** + **LangChain + 通义千问 (Qwen) API** 的技术栈，先跑通核心闭环：**用户输入内容 → AI联网搜索+生成题目 → 用户答题闯关 → AI生成分析报告**。不做用户登录，不做数据库，纯无状态走通核心业务流程。

后端通过 **LangChain 框架**统一对接大模型，而非直接使用 OpenAI SDK。LangChain 提供模型抽象层、结构化输出、Prompt 模板管理等能力，便于后续切换模型、扩展 RAG 等高级功能。

## 二、技术选型（重点对比）

### 2.1 前端框架选型对比

| 维度 | 微信原生开发 | Taro 4 (React) | uni-app |
|------|-------------|----------------|---------|
| **语言** | WXML/WXSS/JS | React + TSX + CSS | Vue + HTML + CSS |
| **TypeScript** | 支持但繁琐 | 原生支持，生态好 | 支持 |
| **学习成本** | 需学WXML/WXS语法 | React开发者零成本 | Vue开发者零成本 |
| **跨端能力** | 仅微信 | H5/RN/多端小程序 | H5/App/多端小程序/鸿蒙 |
| **渲染性能** | 最优，Skyline原生支持 | 良好，Taro 4已适配Skyline | 良好 |
| **组件库** | WeUI | NutUI (京东) / Taro UI | uView / uni-ui |
| **包体积** | 最小 | 有运行时开销(~80KB) | 有运行时开销 |
| **调试体验** | 微信开发者工具原生 | VSCode + 微信开发者工具 + HMR | HBuilderX / VSCode |
| **社区/维护** | 微信官方 | 京东凹凸实验室，活跃 | DCloud，900万开发者 |

**选型结论：Taro 4 (React/TypeScript)**

理由：
1. 熟悉React/TypeScript，**零学习成本**
2. MVP仅需微信小程序，但Taro保留了未来扩展H5/多端的能力
3. Taro 4 性能显著提升，已适配微信 **Skyline 渲染引擎**（更接近原生性能）
4. VSCode开发体验好，NutUI组件库风格适合学习类产品
5. uni-app 是Vue生态，不匹配技术栈；原生开发学习成本高且不可跨端

### 2.2 后端技术选型

FastAPI (Python) 已确认。核心优势：
- **AI生态最强**：LangChain / LangGraph 等原生Python支持
- **Pydantic自动校验**：天然适合处理AI返回的JSON结构，且与LangChain的结构化输出完美搭配
- **异步IO**：适合IO密集的AI调用场景
- **Swagger自动文档**：方便前后端联调

### 2.2.1 AI框架选型：LangChain

使用 **LangChain** 框架对接大模型，而非直接使用 OpenAI SDK。

| 维度 | 直接使用 OpenAI SDK | LangChain 框架 |
|------|-------------------|---------------|
| **模型切换** | 手动改代码，每个模型SDK不同 | `init_chat_model("qwen-plus")` 一行切换 |
| **结构化输出** | 手动解析JSON + 校验 | `with_structured_output(Pydantic模型)` 自动校验+重试 |
| **Prompt管理** | 字符串拼接 | `ChatPromptTemplate` 模板化管理，支持变量注入 |
| **流式输出** | 手动处理SSE | `model.stream()` 统一流式接口 |
| **RAG扩展** | 需从零搭建 | 内置 Document Loader / VectorStore / Retriever |
| **调试追踪** | 无 | LangSmith 可视化追踪每次AI调用 |
| **错误处理** | 手动重试逻辑 | 内置指数退避重试(max_retries)、结构化输出自动纠错 |

**核心依赖包**：
```
langchain          # 核心框架
dashscope          # 通义千问底层SDK（ChatTongyi依赖）
langchain-community # ChatTongyi 集成
```

**关键用法**（基于官方文档确认）：

```python
# 1. 初始化通义千问模型
from langchain_community.chat_models.tongyi import ChatTongyi

llm = ChatTongyi(
    model="qwen-plus",
    streaming=True,
    # enable_search 通过 model_kwargs 传递
)

# 2. 结构化输出：直接输出Pydantic模型，自动校验
from pydantic import BaseModel, Field

class Quiz(BaseModel):
    """AI生成的题目集"""
    topic: str = Field(description="知识主题")
    questions: list[Question] = Field(description="题目列表")

structured_llm = llm.with_structured_output(Quiz)
result: Quiz = structured_llm.invoke(prompt_messages)
# result 已经是校验后的 Pydantic 对象，无需手动解析JSON

# 3. Prompt模板管理
from langchain.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的教育出题专家..."),
    ("human", "请为以下主题生成{count}道闯关题目：{topic}")
])
chain = prompt | structured_llm
result = chain.invoke({"topic": "量子计算", "count": 5})

# 4. 流式输出
for chunk in llm.stream(messages):
    print(chunk.content, end="", flush=True)

# 5. 未来切换模型（几乎零成本）
from langchain.chat_models import init_chat_model
llm = init_chat_model("deepseek-chat", model_provider="openai",
                      base_url="https://api.deepseek.com")
```

### 2.3 AI大模型选型对比（重点）

| 维度 | 通义千问 (Qwen) | DeepSeek V3.2 | 智谱 GLM-4 | Moonshot/Kimi |
|------|----------------|-------------|-----------|--------------|
| **联网搜索** | ✅ `enable_search:true` 原生支持 | ❌ API不支持联网 | ✅ web_search工具 | ✅ 原生支持 |
| **JSON结构化输出** | ✅ `response_format: json_object` | ✅ JSON Mode | ✅ 支持 | ✅ 支持 |
| **API兼容性** | OpenAI兼容格式 | OpenAI兼容格式 | OpenAI兼容 | OpenAI兼容 |
| **价格/百万token** | qwen-plus ≈ ¥4入/¥12出 | ≈ ¥2入/¥8出 | ≈ ¥15/¥15 | ≈ ¥12 |
| **内容安全能力** | 内置安全护栏(DataInspection) | 基础过滤 | 基础过滤 | 基础过滤 |

**选型结论：通义千问 (Qwen-Plus) 为主模型**

关键原因：**联网搜索是核心需求**。通义千问原生支持 `enable_search: true`，一次API调用同时完成"联网搜索知识拓展 + 结构化JSON题目生成"，**无需自建搜索链路**。DeepSeek虽然便宜但API不支持联网搜索，需要额外对接搜索引擎，增加了架构复杂度。

**备选方案**：DeepSeek V3.2 作为降级模型（价格更低，但需自行接入搜索API如SerpAPI）。由于 LangChain 提供统一的模型抽象层，通过 `init_chat_model()` 可一行代码切换模型，切换成本极低。

### 3.2 核心数据流

```
用户输入"量子计算基础"
    ↓
前端 POST /api/v1/generate-quiz { topic: "量子计算基础" }
    ↓
后端 LangChain ChatPromptTemplate 构造Prompt → ChatTongyi.with_structured_output(Quiz) 调用
    ↓
AI联网搜索最新知识 → LangChain自动解析为Pydantic对象(Quiz)
    ↓
结构化输出已自动校验 → 微信内容安全API过滤 → 返回前端
    ↓
前端渲染闯关界面 → 用户逐题作答 → 前端记录答题数据
    ↓
全部答完 → POST /api/v1/generate-report { answers, quiz }
    ↓
AI生成知识总结+复盘报告 → 返回前端
    ↓
前端展示报告 → Canvas生成分享海报 → 保存/转发
```

## 四、核心数据结构

```typescript
// --- 请求 ---
interface GenerateQuizRequest {
  topic: string;           // 用户输入的知识主题
  questionCount?: number;  // 题目数量，默认5
}

// --- 题目 ---
interface Question {
  id: number;
  type: 'single' | 'multiple' | 'judge';
  stem: string;             // 题干
  options: { index: number; content: string }[];
  correctAnswer: number[];  // 正确答案索引
  explanation: string;      // 详细知识讲解
  knowledgePoint: string;   // 关联知识点
  difficulty: 1 | 2 | 3;   // 难度
}

interface Quiz {
  topic: string;
  knowledgeSummary: string; // AI生成的知识概要
  questions: Question[];
  totalXP: number;
}

// --- 答题记录 ---
interface AnswerRecord {
  questionId: number;
  selectedAnswer: number[];
  isCorrect: boolean;
  timeSpent: number;        // 秒
}

// --- 报告 ---
interface Report {
  topic: string;
  correctRate: number;
  totalQuestions: number;
  correctCount: number;
  earnedXP: number;
  earnedCoins: number;
  knowledgeSummary: string;
  weakPoints: string[];
  motivationalQuote: string; // 金句（海报用）
  masteryPercentage: number;
}
```

## 五、核心Prompt设计

### 5.1 出题Prompt（关键）
你是一名专业的 AI 学习教练，请根据用户提供的学习内容生成一组用于小程序闯关答题的题目。

要求：
1. 输出必须是合法 JSON，不要输出任何 JSON 之外的内容。
2. 题目总数为 5 题。
3. 题型包含：3 道单选题、1 道多选题、1 道判断题。
4. 每道题必须包含：题目编号、题型、题干、选项、正确答案、详细讲解、知识点标签、难度。
5. 讲解必须适合初学者阅读，避免过度学术化。
6. 如果用户输入内容过短，可以基于常识进行合理补充，但不要偏离主题。

JSON 输出结构示例：
{
  "title": "学习主题",
  "summary": "本次题库的主题摘要",
  "questions": [
    {
      "id": "q1",
      "type": "single",
      "stem": "题干",
      "options": [
        {"key": "A", "text": "选项A"},
        {"key": "B", "text": "选项B"}
      ],
      "answer": ["A"],
      "explanation": "详细讲解",
      "knowledge_point": "知识点",
      "difficulty": "easy"
    }
  ]
}

用户学习内容如下：
{user_input}

## 六、分步开发计划（核心）

### Phase 0：跑通核心闭环（无登录、无数据库）

目标：以最短路径验证「输入 → 出题 → 答题 → 报告」端到端可行性。

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| **Step 1** | **后端出题接口**：初始化FastAPI项目 → 配置千问API Key → 实现 `POST /generate-quiz`（LangChain ChatPromptTemplate + ChatTongyi.with_structured_output） → 用 Swagger/curl 验证接口可用 | 可独立调用的出题API，输入topic返回结构化Quiz JSON |
| **Step 2** | **前端闯关页**：初始化Taro 4项目 → 先用Mock数据开发闯关页（题目卡片、选项点击、正答/错答反馈、知识讲解展示、XP进度条） → 对接Step 1真实接口联调 | 可完整答题的闯关页面 |
| **Step 3** | **后端报告接口 + 前端报告页**：实现 `POST /generate-report` → 前端开发报告页（正确率、XP、金币、知识总结、薄弱点）→ 报告页底部增加Canvas生成分享海报（金句+成绩+小程序码占位） | 答完题后可查看报告并生成海报 |
| **Step 4** | **首页 + 端到端串联**：开发首页（大输入框 + "开始闯关"按钮 + loading动画）→ 串联 首页→闯关页→报告页 的完整路由跳转 → 接入内容安全（用户输入 msgSecCheck + 千问安全护栏）→ 端到端走通测试 | 完整可体验的MVP闭环 |

### Phase 1：用户体系 + 数据持久化

目标：引入用户身份和数据存储，让用户的学习记录可保存、可回溯。

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| **Step 1** | **微信登录**：实现微信授权登录（`wx.login` → 后端 `code2session` → 返回自定义token）→ 前端封装登录态管理（静默登录，无感知） | 用户可登录，请求携带身份标识 |
| **Step 2** | **数据库设计与接入**：引入 PostgreSQL + SQLAlchemy → 设计核心表（用户表、闯关记录表、答题明细表）→ 改造出题/报告接口，答题数据落库 | 每次闯关数据持久化存储 |
| **Step 3** | **个人中心**：开发个人中心页面 → 历史闯关记录列表（支持点击查看历史报告）→ 累计经验值/金币/闯关次数统计 | 用户可查看历史学习数据 |
| **Step 4** | **答题分析图表**：个人中心增加数据看板 → 正确率趋势图、知识领域分布图、薄弱知识点排行 | 可视化学习分析 |

### Phase 2：输入源拓宽 + 学习增强

目标：从"一句话出题"扩展到多种输入源，增强学习留存机制。

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| **Step 1** | **长文本输入**：首页输入框支持多行长文本 → 后端增加Prompt适配（从长文本中提取知识点再出题）→ 前端增加字数提示 | 支持粘贴一段文章直接出题 |
| **Step 2** | **文档上传**：前端支持上传PDF/Word → 后端接入文档解析（LangChain Document Loader / PyPDF / python-docx）→ 提取文本后走出题链路 | 支持上传文档自动出题 |
| **Step 3** | **URL解析**：前端支持输入网页URL → 后端接入网页抓取（LangChain WebBaseLoader / Firecrawl）→ 提取正文后出题 | 支持输入网址自动出题 |
| **Step 4** | **错题重练**：记录用户答错的题目 → 结合艾宾浩斯遗忘曲线计算复习时间 → 到期时在首页提示"旧识重温"关卡 | 错题自动复习机制 |

### Phase 3：社交裂变 + 游戏化增强

目标：通过社交功能和游戏化机制提升用户活跃与传播。

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| **Step 1** | **分享增强**：优化海报样式 → 海报嵌入真实小程序码 → 好友扫码直接进入相同主题闯关 | 社交裂变入口 |
| **Step 2** | **好友PK**：支持邀请微信好友进行同题异步PK → 双方答完后展示对比结果（正确率、用时、XP） | 双人PK闯关玩法 |
| **Step 3** | **排行榜**：按知识领域划分排行榜 → 展示每周/总榜 → 个人排名与击败百分比 | 全服排行榜系统 |
| **Step 4** | **成就系统**：设计成就徽章（首次闯关、连续答对、累计XP等）→ 个人中心展示成就墙 | 游戏化成就激励 |

### Phase 4：RAG知识库 + 商业化

目标：支持私有知识库出题，探索商业变现模式。

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| **Step 1** | **RAG知识库**：引入向量数据库（Chroma）→ 用户可创建私有知识库文件夹 → 上传文档自动向量化 → 出题时从知识库检索相关内容（LangChain Retriever + RAG Chain） | 私有知识库出题能力 |
| **Step 2** | **体力值 + 免费额度**：引入每日免费闯关次数限制 → 超出消耗"体力值"→ 体力值可通过看广告恢复或付费购买 | Freemium基础变现 |
| **Step 3** | **VIP订阅**：推出VIP套餐（无限闯关、大文档上传、导出知识图谱）→ 微信支付接入 | 订阅制收入 |
| **Step 4** | **B端定制**：提供企业版API → 支持企业上传内部培训资料生成题库 → 管理后台查看员工答题数据 | B2B定制套餐 |

## 九、关键技术要点

### 9.1 AI输出可靠性（LangChain结构化输出）
- 使用 `llm.with_structured_output(PydanticModel)` 替代手动 `response_format: json_object`
- LangChain 自动将AI输出解析为 Pydantic 对象，校验失败时自动反馈错误信息给模型并重试
- 支持 `include_raw=True` 同时获取原始AI消息用于调试
- 底层优先使用 `function_calling` 方式，兼容性最广
- 兜底：`max_retries` 参数控制网络层重试，结构化输出自动纠错处理模型格式错误

### 9.2 等待体验优化
- AI出题约3-10秒，设计引导动画（"AI正在联网学习中..."动态文案）
- 可选：流式返回（增加复杂度，Phase 0暂不做）

### 9.3 经验值计算规则
- 答对：XP = 10 × 难度系数(1/1.5/2)
- 答错：XP = 0
- 金币：通关后 coins = 正确题数 × 2

### 9.4 内容安全双保险
1. 用户输入 → 微信 `msgSecCheck` API
2. AI输出 → 千问 `X-DashScope-DataInspection` 安全护栏
3. 兜底 → 前端基础敏感词过滤

### 9.5 海报生成
- Taro Canvas API 绘制模板 → `canvasToTempFilePath` 导出 → `saveImageToPhotosAlbum` 保存

## 十、风险与应对

| 风险 | 应对 |
|------|------|
| AI生成JSON格式不稳定 | LangChain `with_structured_output` 自动校验+纠错重试 |
| AI生成时间过长(>10s) | 引导动画 + 超时设置 + 减少题数 |
| AI生成不当内容 | 双重安全过滤（千问+微信API） |
| 千问API故障 | LangChain `init_chat_model()` 一行切换备选DeepSeek模型 |
| 小程序审核被拒 | 提前了解AI内容审核要求，确保安全过滤到位 |
